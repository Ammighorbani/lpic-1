# at

### You can calculate your jobs like `cron` but when you need your job run only one time not be in a time base loop with `at`

---

### How to set job with at?

```bash
at [time]

at 16:38
```

**After that you can write your commands and then use `CTRL + d` to exit and confirm your job**

---

### You can set job in AM or PM in specefic days

```bash
at [time] [AM-PM] [day-of-week]

at 16:30 AM Mon
```

---

### You can set job in specefic dates

```bash
at [time] [AM-PM] [month] [day-of-month]

at 16:30 PM July 27
```

### Note: If you run this type of at in July 28 and you say July 27 your job will start next year in July 27

---

### You can set job in specefic day month year

```bash
at [time] [AM-PM] [month]/[day]/[year]

at 16:30 PM 8/23/2028
```

---

### You can say go to next dates

```bash
at 16:30 PM next month
```

```bash
at 16:30 PM next year
```

```bash
at 16:30 PM tomorrow
```

```bash
at teatime
```

### Note: `teatime` is 16:00

```bash
at midnight
```

### Note: `midnight` is 00:00

```bash
at now + 1 week
```

```bash
at now + 6 year
```

---

### How to see list of at

```bash
atq
```

---

### How to remove a job?
**First you need to give list of jobs and now the job id and then use below command**
```bash
atrm [id]

atrm 3
```

---

### How to see each job have which commands

```bash
at -c [id]

at -c 3
```

---

### How to give access from users to don't make at and only specefic user can write at job?

**For first you need to create a file in `/etc` with `at.allow` name**

```bash
vim /etc/at.allow
```

**After you created this file you can write that username you want to have access to write at for you**

```bash
root
```

### Note: After you make this file whole users access will remove from at and only that user can create at job you wroted it name in that file

---

### How to give access from specefic user to don't make at?

**For first you need to create a file in `/etc` with `at.deny` name**

```bash
vim /etc/at.deny
```

**After you created this file you can write that username you want to don't have access to write at for you**

```bash
ammi
```

### Note: After you make this file that specefic users access will remove from at and they can't create at job
