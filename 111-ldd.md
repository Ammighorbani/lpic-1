# ldd

### ldd is command for you undrestand your services are using which `shared library`

### Note: When you want to use `ldd` command you need to know the absolute path of that service [read more](110-which-whereis.md)


```bash
ldd /usr/bin/ls
```

---

### Note: `ldd` command don't have `shared library` and whole library it use is `static library` and it have theme in it own package

---

### Where is `ldd` command `static library`?
**You can find `ldd` command `static library` in this route `/etc/ld.so.conf` and in that file transfer you to another file with `/etc/ld.so.conf.d/*.conf` route and it's mean go into that route and check there if you find any files with `.conf` extention read it and other `shared library` routes are there**

### Note: Why in ldd config file transfered you into `/etc/ld.so.conf.d/*.conf`?
**This transfer is because if oneday you needed to write and custom your own `shared library` your `OS` read your own config files either you can create a file with `.conf` extention look like `local.conf` and put your own shared library routes there**

---

### If everytime you run a command your command execute too fast if your OS need everytime use ldd to find shared library routes?
**This part of story is one of the most intresting parts, each time your `OS` turn on it will one time path through whole this directory to directory story and after finding whole `shared library` routes it will catch it for you to don't need do it every single time and you can see your catch file in `/etc/ld.so.catch` route**

---

### Note: You can see your catched routes file better and more clear with this command

```bash
ldconfig -v
```
